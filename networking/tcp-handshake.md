# TCP Three-Way Handshake

TCP uses a three-way handshake before sending data.

## Step 1 - SYN

The client requests a connection.

## Step 2 - SYN-ACK

The server acknowledges the request and responds.

## Step 3 - ACK

The client confirms the connection.

## Why TCP Uses a Handshake

- Ensures both devices are reachable.
- Establishes a reliable connection.
- Verifies both sides are ready for communication.

## TCP vs UDP

### TCP

- Reliable
- Connection-oriented
- Uses a three-way handshake

### UDP

- Faster
- Connectionless
- No handshake

## What I Learned

- TCP establishes a connection before transmitting data.
- UDP does not perform a handshake.
- Reliability is one of the key differences between TCP and UDP.
