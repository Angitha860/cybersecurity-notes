# Hashing vs Encryption

Hashing and Encryption are two important security concepts.

## Hashing

Hashing converts data into a fixed-length value called a hash.

Example:

Password123 → Hash

Characteristics:

- One-way process
- Cannot be easily reversed
- Used for password storage
- Used for integrity verification

## Encryption

Encryption converts readable data into unreadable data.

Example:

Message → Encrypted Data

Characteristics:

- Reversible using a key
- Protects confidentiality
- Used in HTTPS, VPNs and secure messaging

## Hashing vs Encryption

| Hashing | Encryption |
|----------|------------|
| One-way | Reversible |
| Used for Integrity | Used for Confidentiality |
| Password Storage | Secure Communication |

## CIA Triad Connection

- Hashing helps protect Integrity.
- Encryption helps protect Confidentiality.

## What I Learned

- Hashing is one-way.
- Encryption can be reversed using a key.
- Passwords should be hashed.
- Secure communications use encryption.
