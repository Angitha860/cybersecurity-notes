# Indicators of Compromise (IOC)

An Indicator of Compromise (IOC) is evidence that suggests a system, account, or network may have been compromised.

IOCs help security analysts detect and investigate potential security incidents.

## Common IOC Types

### Network IOCs

Examples:

- Malicious IP addresses
- Suspicious domains
- Unusual DNS requests
- Unexpected network connections

### Host IOCs

Examples:

- Suspicious processes
- Modified files
- Unauthorized software
- New user accounts

### User IOCs

Examples:

- Impossible travel logins
- Multiple failed logins
- Unusual login times
- Login attempts from unfamiliar locations

## Examples

### Suspicious IP Address

A device communicates with a known malicious IP address.

### Suspicious Domain

A user visits a phishing website.

### Suspicious Process

An unknown executable starts running on a workstation.

### Impossible Travel

A user logs in from two distant countries within a short period of time.

## Important Note

An IOC does not always prove an attack occurred.

An IOC is evidence that requires further investigation.

## What I Learned

- IOC stands for Indicator of Compromise.
- IOCs help identify possible security incidents.
- IOCs can be network, host, or user related.
- Analysts investigate IOCs before confirming an attack.
