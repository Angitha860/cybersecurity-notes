# Incident Response Lifecycle

Incident Response is a structured process used to handle security incidents.

## Phases

### 1. Preparation

Activities performed before incidents occur.

Examples:

- Security policies
- Backups
- Antivirus
- SIEM
- Incident response plans

### 2. Identification

Determine whether a security incident has occurred.

Examples:

- Investigating alerts
- Confirming ransomware
- Identifying affected systems

### 3. Containment

Limit the spread and impact of the incident.

Examples:

- Disconnect infected systems
- Disable compromised accounts
- Isolate endpoints

### 4. Eradication

Remove the threat from the environment.

Examples:

- Remove malware
- Delete malicious accounts
- Apply security patches

### 5. Recovery

Restore systems to normal operations.

Examples:

- Restore backups
- Reconnect systems
- Resume business operations

### 6. Lessons Learned

Review the incident and improve security controls.

Examples:

- Root cause analysis
- Security awareness training
- Updating procedures

## Incident Response Flow

Preparation
↓
Identification
↓
Containment
↓
Eradication
↓
Recovery
↓
Lessons Learned

## What I Learned

- Incident response follows a structured process.
- Containment aims to stop the spread of threats.
- Recovery restores business operations.
- Lessons learned help prevent future incidents.
